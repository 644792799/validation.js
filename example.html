Simple Form Validation

Prototypal form validation. Full docs coming soon.

Tested in the following browsers:

  • Firefox 3.6
  • Firefox 4
  • Safari 5
  • Chrome 8 and 9
  • Internet Explorer 7, 8 and 9

Installation

Include the Javascript file (minified version recommended for production use) within your page and instantiate a new validation object using the Validate() constructor method. The constructor takes 3 arguments:-

  • Form ID {string}
  • Validation methods {object}
  • Options {object} (Not required)

	var check_this = Validate('my_form', { methods }, { options });

The second argument is an object containing your validation methods in the format input_id: { validation_method: argument } and accompanying error message. Current methods available are:-

  • present {boolean} A value must or must not be present
  • test {string} Reference to regular expression
  • minimum_length {integer} A string is longer than given characters
  • maximum_length {integer} A string is less than given characters
  • greater_than {integer} A number is greater than
  • less_than {integer} A number is less than

NOTE: Currently you can only attach a single error message to each form input so choose your words wisely.

Options can optionally be passed into each instance:-

  • error_node {string} (Default: div) Node to wrap error message with
  • error_class {string} (Default: form_error) Class to apply to error node
  • error_display {boolean} Default: true Display errors (you can always retrieve errors manually)
  • error_placement {string} before|after Show errors at the top or bottom of the inputs parent node (before|after)
